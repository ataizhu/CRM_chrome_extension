# CRM Tasks Manager - Chrome Extension

Chrome расширение для управления задачами с боковым меню, группировкой и контекстным меню в стиле TabAI.

## Возможности

- 📋 Боковое меню (Side Panel) вместо popup
- ➕ Добавление задач через контекстное меню (выделить текст → ПКМ)
- 🏷️ Группировка задач по категориям
- ✏️ Редактирование задач
- ✅ Отметка задач как выполненных
- 🗑️ Удаление задач
- 💾 Синхронизация через `chrome.storage.sync`
- 🔄 Автоматическая синхронизация с API
- 🎨 Современный дизайн в стиле TabAI

## Установка

1. Откройте Chrome и перейдите в `chrome://extensions/`
2. Включите "Режим разработчика"
3. Нажмите "Загрузить распакованное расширение"
4. Выберите папку с расширением

## Использование

### Боковое меню
- Кликните по иконке расширения в панели инструментов
- Или используйте контекстное меню для быстрого добавления задач

### Контекстное меню
1. Выделите любой текст на странице
2. Правой кнопкой мыши → "Добавить задачу: [текст]"
3. Боковое меню откроется автоматически с заполненной формой

### Группировка
- Создавайте группы через кнопку "+" в секции "Группы"
- Выбирайте группу при добавлении задачи
- Фильтруйте задачи по группам, кликая на чипсы групп
- Задачи автоматически группируются при отображении

## Настройка API

1. Откройте боковое меню
2. Нажмите на иконку настроек (⚙️)
3. Введите URL вашего API
4. Нажмите "Сохранить"

## Формат API

Расширение ожидает REST API со следующими endpoints:

- `GET /tasks` - получить список задач
- `POST /tasks` - создать задачу (body: `{ text: string, completed: boolean, group?: string }`)
- `PUT /tasks/:id` - обновить задачу (body: `{ text?: string, completed?: boolean, group?: string }`)
- `DELETE /tasks/:id` - удалить задачу

Формат задачи:
```json
{
  "id": "1",
  "text": "Текст задачи",
  "completed": false,
  "group": "Работа"
}
```

## Примечания

- Если API недоступен, расширение работает с локальными данными из `chrome.storage.sync`
- Данные автоматически синхронизируются при каждом действии
- Расширение работает офлайн с локальными данными
- Группы сохраняются локально и синхронизируются через storage
