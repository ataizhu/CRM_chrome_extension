# Хранение данных в расширении CRM Tasks

## Где и что сохраняется

### 1. **Личные задачи** (`chrome.storage.sync`)
- **Ключ:** `personalTasks`
- **Тип:** массив задач
- **Синхронизация:** Да, между устройствами через аккаунт Google
- **Лимит:** 100KB на элемент (примерно 1000-2000 задач в зависимости от размера)

### 2. **Удаленные CRM задачи** (`chrome.storage.sync`)
- **Ключ:** `deletedCrmTasks`
- **Тип:** массив ID задач (только строки/числа)
- **Синхронизация:** Да, между устройствами
- **Лимит:** 100KB на элемент (примерно 10,000-50,000 ID)

### 3. **CRM задачи** (НЕ сохраняются локально)
- **Источник:** загружаются из API каждый раз при открытии расширения
- **Хранение:** только в памяти во время работы расширения
- **Безопасность:** данные всегда актуальные, но требуют интернет

### 4. **Авторизация** (`chrome.storage.local`)
- **Ключи:** `authToken`, `user`, `vtigerCredentials`
- **Тип:** объекты с данными авторизации
- **Синхронизация:** НЕТ (только на текущем устройстве)
- **Безопасность:** пароли хранятся в открытом виде (но только локально)

### 5. **Настройки** (`chrome.storage.sync`)
- **Ключи:** `theme`, `selectedGroup`, `syncPeriod`, `authMode`, `visibleGroups`, `groups`
- **Синхронизация:** Да, между устройствами

## Ограничения Chrome Storage

### `chrome.storage.sync`
- **100KB на элемент** (один ключ)
- **100KB общий лимит** (все ключи вместе) - для бесплатных аккаунтов
- **8KB на элемент** - для бесплатных аккаунтов (если превышен общий лимит)
- **При превышении:** Chrome автоматически переключается на `chrome.storage.local`

### `chrome.storage.local`
- **10MB общий лимит** (все расширения вместе)
- **Без синхронизации** между устройствами
- **Данные не удаляются** автоматически (только при удалении расширения)

## Безопасность данных

### ✅ Что безопасно:
1. **Личные задачи** - синхронизируются между устройствами, хранятся в зашифрованном виде на серверах Google
2. **Настройки** - синхронизируются безопасно
3. **CRM задачи** - не хранятся локально, всегда актуальные из API

### ⚠️ Потенциальные риски:
1. **Превышение лимита sync (100KB)** - Chrome может переключиться на `local`, тогда синхронизация пропадет
2. **Удаление расширения** - все данные (`sync` и `local`) будут удалены
3. **Очистка данных браузера** - если пользователь очистит данные расширения, все пропадет
4. **Пароли в `local`** - хранятся в открытом виде (но только на устройстве пользователя)

## Рекомендации

### Для предотвращения потери данных:

1. **Регулярная очистка удаленных CRM задач:**
   - Если удалено много задач, список `deletedCrmTasks` может расти
   - Можно периодически очищать старые записи

2. **Мониторинг размера данных:**
   - Проверять размер `personalTasks` перед сохранением
   - Если задач много, можно архивировать старые

3. **Резервное копирование:**
   - Можно добавить экспорт задач в JSON файл
   - Или синхронизацию с внешним сервисом

4. **Для CRM задач:**
   - Данные всегда в безопасности, так как хранятся на сервере
   - Локальное удаление не влияет на сервер

## Текущее состояние

Сейчас расширение использует:
- `chrome.storage.sync` для личных задач и настроек (синхронизация между устройствами)
- `chrome.storage.local` для авторизации (безопасность, не синхронизируется)
- API для CRM задач (всегда актуальные данные)

**Вывод:** Данные достаточно безопасны, но при большом количестве задач (>1000-2000) может возникнуть проблема с лимитом `chrome.storage.sync`.
