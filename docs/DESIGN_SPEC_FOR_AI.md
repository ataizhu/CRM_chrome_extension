# Спецификация приложения CRM Tasks — для AI-дизайна

Chrome-расширение (Side Panel). Управление задачами: CRM (из API) + личные (локально). Язык интерфейса — русский.

---

## 1. ОБЩАЯ СТРУКТУРА

- **Контейнер:** `.app` — flex column, 100vh, вся ширина.
- **Верхняя панель:** фиксированная шапка (top bar).
- **Основная область:** 3 вкладки (табы) — переключение без отдельной навигации, через кнопки в шапке:
  - **Задачи** (по умолчанию)
  - **Настройки**
  - **Статистика**

---

## 2. ВЕРХНЯЯ ПАНЕЛЬ (HEADER / TOP BAR)

Горизонтальная полоса: `backdrop-blur`, скруглённые углы (16px), тень, border. Слева направо:

### 2.1 Кнопка «Новая задача» (`plusBtn`)
- **Вид:** иконка «+» (24×24), квадратная кнопка ~40×40, скругление 10px.
- **Действие:** открывает модальное окно создания задачи.
- **Hover:** лёгкий scale + сдвиг вверх, фон `--bg-secondary`.

### 2.2 Дропдаун «Группы» (`groupDropdownTrigger` + `groupDropdownPanel`)
- **Триггер:** показывает текущую выбранную группу (напр. «CRM», «Личные») + шеврон вниз. Min-width ~120px, скругление 10px.
- **При клике** открывается выпадающая панель под триггером.

**Внутри панели:**
- Заголовок «Выберите опцию» + кнопка «Добавить группу» (иконка +).
- Строка добавления группы (скрыта по умолчанию): поле «Название группы» + кнопка «Добавить».
- Поле поиска с иконкой лупы, placeholder «Искать».
- Список групп: каждая строка — **чекбокс** + название. Чекбоксы управляют **видимыми** группами (какие показывать в списке задач). Выбор группы для фильтра — по клику на строку (не только по чекбоксу).

**Важно:** несколько групп можно выбрать галочками; в основной области отображаются только выбранные. Группы можно добавлять динамически.

### 2.3 Кнопка «Настройки» (`settingsBtn`)
- Иконка шестерёнки, ~40×40, скругление 10px.
- **Действие:** переключает на вкладку «Настройки» (если уже там — обратно на «Задачи»).

### 2.4 Кнопка «Статистика» (`statsBtn`)
- Иконка столбчатой диаграммы (три вертикальные полоски), ~40×40.
- **Действие:** переключает на вкладку «Статистика» (если уже там — обратно на «Задачи»).

---

## 3. ВКЛАДКА «ЗАДАЧИ» (`tabTasks`)

### 3.1 Состояния области контента
- **Загрузка:** блок `#loading` — спиннер + текст «Загрузка...», по центру.
- **Ошибка:** блок `#error` — красноватый фон, border, текст ошибки.
- **Нет задач:** `empty-state` — иконка 📋, текст «Нет задач» или «Нет задач в выбранных группах».
- **Есть задачи:** список **групп** с задачами внутри.

### 3.2 Группы задач (`task-group`)
- Карточка: фон `--bg-primary`, border, скругление ~10px, тень.
- **Шапка группы (`group-header`):**
  - Название группы (например «CRM», «Личные») + счётчик задач в бейдже.
  - Справа — кнопка-шеврон (развернуть/свернуть список задач).
- **Тело (`group-tasks`):** список карточек задач. По клику на шапку или шеврон — сворачивание/разворачивание.

### 3.3 Карточка задачи (`task-item`)
**Основная строка (`task-row`):**
- Чекбокс (выполнено/не выполнено).
- Текст задачи (`task-text`), переносится, может занимать несколько строк.
- Справа **фиксированный блок** (`task-meta`):
  - **Статус:** либо иконка, либо время.
    - **Иконки статуса** (цветные, с `data-tooltip` при наведении):
      - **Выполнено** — зелёная.
      - **Просрочено** — красная.
      - **Без срока** — серая.
    - **Время:** бейдж «X дн» / «X ч» / «X мин» (оставшееся до дедлайна).
  - Кнопка **удаления** (иконка корзины), красноватый фон, при hover — заливка красным.

**Раскрытая карточка (`task-detail`)** — по клику по строке (не по чекбоксу/удалению):
- Две строки: **Начало** | **Окончание** (дата и время).
- **Описание:** заголовок «Описание» по центру, текст на всю ширину.
- Кнопка **«Редактировать»** внизу — открывает ту же модалку создания/редактирования с заполненными полями.

**Состояния:**
- `completed`: зачёркнутый текст, пониженная opacity.
- `expanded`: показывается `task-detail`, усиленная тень/border.

## 4. ВКЛАДКА «НАСТРОЙКИ» (`tabSettings`)

Вертикальный скролл. Секции — карточки с заголовком в стиле «SETTINGS» (мелкий uppercase, letter-spacing).

### 4.1 Блок «Авторизация» (`authSection`)
**Если не авторизован:**
- Форма: **Логин** (текст), **Пароль** (пароль).
- Текст подсказки (`loginHint`): «Логин и пароль как при входе в веб-интерфейс Vtiger.»
- Блок `#loginError` для ошибок (скрыт по умолчанию).
- Кнопка **«Войти»** (primary).

**Если авторизован:**
- Блок `authStatus`: иконка-галочка в зелёном круге, подпись «Авторизован», имя пользователя, кнопка **«Выйти»**.

### 4.2 Блок «Синхронизация CRM» (`syncSection`)
- Показывается **только после авторизации**.
- Подсказка: «Выберите период и загрузите задачи.»
- **Select «Период»:** Сегодня | Неделя | Месяц | Всё время.
- Кнопка **«Загрузить задачи»** (primary).
- Блок `#syncStatus` для результата (успех/ошибка), скрыт по умолчанию.

### 4.3 Блок «Внешний вид»
- Заголовок «Тема».
- Две кнопки-переключателя: **«Светлая»** | **«Тёмная»**. Активная — accent-цвет, неактивная — нейтральный.

### 4.4 Блок «Управление задачами»
- Одна кнопка: **«Очистить все задачи»** (danger) + иконка корзины.
- **Действие:** открывается **модальное окно выбора групп** (см. п. 6.2). После выбора групп и подтверждения — очистка выбранных (CRM — локальное «удаление», личные — удаление из хранилища).

### 4.5 Блок «Уведомления о задачах» (`notifySection`)
- Показывается **только после авторизации**.
- Чекбокс «Включить уведомления».

---

## 5. ВКЛАДКА «СТАТИСТИКА» (`tabStats`)

- **Тулбар:** один `select` с периодом (Сегодня | Неделя | Месяц | Всё время).
- **Контент (`statsContent`):**
  - **Карточки (stat-cards):** сетка 2×2 — «Всего задач», «Выполнено», «В процессе», «Выполнено %».
  - **Блок «По группам»:** круговая диаграмма (SVG) + легенда с цветными точками и названиями групп.
  - **Блок «Активность по дням»:** столбчатая диаграмма (последние 14 дней), подписи под столбцами.
- **Пусто:** «Нет данных за выбранный период» + иконка 📊.

## 6. МОДАЛЬНЫЕ ОКНА

### 6.1 Модалка «Новая задача» / «Редактировать задачу» (`taskFormModal`)
- **Overlay:** полупрозрачный тёмный фон, по центру модалка.
- **Шапка:** заголовок «Новая задача» или «Редактировать задачу» + кнопка закрытия «✕».

**Форма (используемые поля):**
- **Тема \*** — текстовое поле, обязательно.
- **Группа** — дропдаун (триггер «CRM» / «Личные» и т.д. + панель со списком групп, поиск). Аналог `groupDropdown`, но для выбора группы при создании.
- **Начало** — виджет даты/времени (только для группы CRM). Календарь + часы:минуты.
- **Окончание** — такой же виджет, всегда.
- **Описание** — textarea.

**Режим «CRM»** (группа = CRM): показываются «Начало», «Окончание». Режим «Личные» — упрощённая форма (только тема, группа, окончание, описание); «Начало» скрыто. В HTML также есть блоки «Приоритетная задача», «Тип действия», «Статус», «Ответственный», «Контрагенты» — они в виде дропдаунов/чекбоксов, но в текущей логике не участвуют; можно оставить под будущее использование или убрать при редизайне.

**Низ формы:**
- Кнопки **«Отмена»** (secondary) и **«Создать»** / **«Сохранить»** (primary).

**Виджет даты/времени (dt-picker):**
- Триггер: иконка календаря + текст «Выберите дату и время» или выбранное значение.
- Popover: календарь (месяц, навигация ‹ ›), сетка дней, блок «часы : минуты» (числовые инпуты).

### 6.2 Модалка «Выберите группы для очистки» (при «Очистить все задачи»)
- **Шапка:** заголовок + «✕».
- **Тело:** список групп. Каждая строка — чекбокс + название группы + количество задач (например «5 задач»).
- **Футер:** кнопки **«Отмена»** (secondary) и **«Очистить»** (danger). По «Очистить» удаляются задачи только из выбранных групп.

### 6.3 Модалка «Вход» (`loginModal`)
- Используется редко (основной вход — в настройках). Те же поля: логин, пароль, кнопка «Войти», подсказка, блок ошибок.

## 7. ДИЗАЙН-ТОКЕНЫ (CSS-переменные)

**Светлая тема:**
- Фон: `--bg-primary` #fff, `--bg-secondary` #f5f6f8, `--bg-tertiary` #eef0f3.
- Текст: `--text-primary` #1a1a1a, `--text-secondary` #5c6470, `--text-tertiary` #9ca3af.
- Акцент: `--accent` #3b82f6, hover #2563eb.
- Границы: `--border` #e5e7eb.
- Успех / опасность / предупреждение: `--success`, `--danger`, `--warning`.
- Радиусы: 10px (карточки, инпуты), 8px для мелких элементов.
- Тени: `--shadow-sm`, `--shadow-md`, `--shadow-lg`.

**Тёмная тема:**
- Фоны тёмные (#0a0a0a, #141414, #1e1e1e).
- Текст светлый. Акцент напр. #00bfff.
- Тот же набор переменных, другие значения.

**Кнопки (dock-style):**
- Фон `--dock-bg`, бордер `--dock-border`, при hover — `--dock-bg-hover`, `--dock-border-hover`.
- Primary: заливка accent. Secondary: нейтральный фон. Danger: красноватый фон/текст.

## 8. ИНТЕРАКЦИИ И ПОВЕДЕНИЕ

- **Чекбокс задачи:** переключение «выполнено / не выполнено»; для CRM при необходимости синхронизация с API.
- **Клик по строке задачи** (не по чекбоксу/удалению): раскрыть/свернуть детали.
- **Удаление (иконка корзины):** подтверждение «Удалить задачу?»; для CRM — только локальное «удаление» в расширении.
- **Редактировать:** открыть модалку формы с подставленными данными, кнопка «Сохранить».
- **Группы в дропдауне:** чекбоксы — видимость групп в списке; клик по строке — выбор активной группы и закрытие панели.
- **Вкладки:** переключение через кнопки Настройки / Статистика в шапке (toggle: если уже открыт тот же таб — возврат на Задачи).

---

## 9. ТЕХНИЧЕСКИЕ ОГРАНИЧЕНИЯ

- **Side Panel** Chrome: узкая ширина (обычно 320–400px), вертикальный скролл. Горизонтального скролла избегать.
- Всё должно работать в светлой и тёмной темах.
- Шрифт: системный стек (-apple-system, BlinkMacSystemFont, Segoe UI, Roboto и т.д.), 14px базовый, антиалиасинг.

---

## 10. ЧЕКЛИСТ ДЛЯ AI-ДИЗАЙНА

1. **Top bar:** плюс, дропдаун групп (с чекбоксами и поиском), Настройки, Статистика.
2. **Задачи:** группы с заголовком и счётчиком, сворачивание; карточки задач с чекбоксом, текстом, статусом/временем и корзиной справа; раскрытие деталей и кнопка «Редактировать».
3. **Настройки:** Авторизация (форма / статус), Синхронизация CRM, Тема, Управление задачами (очистка), Уведомления.
4. **Статистика:** период, 4 карточки, круговая диаграмма по группам, столбцы по дням.
5. **Модалки:** форма задачи (в т.ч. дата/время, упрощённый/полный режим), выбор групп для очистки, при необходимости — вход.
6. **Состояния:** загрузка, ошибка, пустые списки, активная тема.
7. **Стиль:** аккуратный, современный UI (dock-like кнопки, карточки, скругления, тени), без перегруженности — под узкий Side Panel.
