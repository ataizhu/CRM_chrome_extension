# Статус рефакторинга sidepanel.js

**Статус: завершён.** Монолитный `sidepanel.js` разнесён по модулям в `js/`.
`sidepanel.js` остался тонкой точкой входа: импортирует модули, собирает объект
зависимостей (`getDependencies`) и инициализирует приложение в `DOMContentLoaded`.

## Текущие модули (`crm/js/`)

| Модуль | Ответственность |
|--------|-----------------|
| `config.js` | Константы и глобальное состояние + сеттеры |
| `storage.js` | `chrome.storage` (личные задачи, группы, авторизация, кэш CRM) |
| `utils.js` | DOM-хелперы, `escapeHtml`, пикер-модалки |
| `auth.js` | Авторизация (login/logout/checkAuth) |
| `api.js` | HTTP-запросы к `extensionAPI_v1.php` |
| `groups.js` | Группы: загрузка, видимость, переименование, drag-reorder |
| `tasks.js` | Загрузка/создание/обновление/удаление задач, merge, заметки |
| `render.js` | Рендеринг списка задач и карточек |
| `ui.js` | События, форма задачи, настройки, плеер записей |
| `datetime-picker.js` | Компонент выбора даты/времени |
| `modals.js` | Модалки (форма задачи, выбор группы, сегменты) |
| `stats.js` | Статистика по CRM-задачам (вкладка в сайдпанели) |
| `changelog.js` + `changelog-data.js` | Окно «Что нового» (генерируется из `changelog-data.json`) |

## Известный технический долг

- `ui.js` всё ещё крупный (~2.7k строк). Запланирован отдельный распил на
  `recording-player.js`, `settings-sync-groups.js`, `task-form.js` (Фаза 2.1 плана).
- Логика расчёта `date_from` по периоду продублирована в `tasks.js`, `api.js`,
  `background.js` (background не может импортировать ES-модули клиента).
