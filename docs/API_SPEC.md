# Спецификация API для CRM-расширения

Расширение ожидает REST API для авторизации и CRUD задач CRM.

## Авторизация

- **URL**: настраивается в расширении (Endpoint авторизации).
- **Метод**: `POST`
- **Headers**: `Content-Type: application/json`
- **Body**: `{ "username": "...", "login": "...", "password": "..." }`  
  (отправляются и `username`, и `login` с одним значением для совместимости с разными API)
- **Успех (200)**:
  - В теле ответа должен быть токен в одном из полей: `token`, `access_token`, `accessToken`, `jwt`.
  - Опционально: `user` — объект с `username` или `email` для отображения в настройках.
- **Ошибка (4xx/5xx)**: JSON с полем `message` или `error` (показывается пользователю).

Далее все запросы к API задач отправляются с заголовком:
```
Authorization: Bearer <token>
```
При ответе **401** расширение сбрасывает сессию и показывает форму входа.

## Задачи CRM (API URL)

- **Базовый URL**: настраивается в расширении (API URL).

### Список задач

- **Метод**: `GET`
- **Query** (опционально):
  - `period=today|week|month|3months|6months|year|all` — период синхронизации.
  - `date_from=YYYY-MM-DD` — дата начала периода. Бэкенд должен вернуть все задачи с датой >= `date_from` **без верхней границы** (включая будущие).
    Фильтрация на сервере должна проверять поля `date_end`, `date_start` или дату создания задачи.
  Если `date_from` не поддерживается, расширение дополнительно фильтрует ответ на клиенте по `end`/`start`/`createdAt`.
- **Успех (200)**:
  - Либо массив задач: `[{ ... }, ...]`
  - Либо объект с массивом: `{ "tasks": [...] }` или `{ "items": [...] }`
- **Поля задачи** (расширение использует):
  - `id` — строка или число
  - `text` — тема/название
  - `completed` — boolean
  - `group` — при получении игнорируется, в UI всегда «CRM»
  - `start`, `end` — ISO 8601 (например `2025-01-15T10:00:00.000Z`)
  - `description` — описание
  - `createdAt` — ISO 8601 (если нет, подставляется текущее время)

### Создание

- **Метод**: `POST`
- **Headers**: `Content-Type: application/json`, `Authorization: Bearer <token>`
- **Body**: `{ "text": "...", "completed": false, "group": "CRM", "start?", "end?", "description?" }`
- **Успех (200/201)**: объект созданной задачи (формат как в списке).

### Обновление

- **Метод**: `PUT`
- **URL**: `{API_URL}/{id}`
- **Body**: `{ "text"? , "group"? , "start"? , "end"? , "description"? }` или `{ "completed": true|false }` (для переключения выполнения).
- **Успех (200)**: объект обновлённой задачи.

### Удаление

- **Метод**: `DELETE`
- **URL**: `{API_URL}/{id}`
- **Успех**: 200/204 или пустое тело.

## Резюме

| Действие     | Метод | URL              | Примечание                    |
|-------------|-------|------------------|-------------------------------|
| Вход        | POST  | Auth endpoint    | `username`/`login` + `password` |
| Список CRM  | GET   | API URL          | `?period=...&date_from=YYYY-MM-DD` |
| Создать     | POST  | API URL          |                               |
| Обновить    | PUT   | API URL / :id    |                               |
| Удалить     | DELETE| API URL / :id    |                               |

Личные задачи (Личные и пользовательские группы) **не** отправляются в API и хранятся только локально в `chrome.storage.sync`.
