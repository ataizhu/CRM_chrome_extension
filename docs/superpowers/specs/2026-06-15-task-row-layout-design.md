# Дизайн: двухъярусная раскладка строки задачи (мета во всю ширину)

- **Дата:** 2026-06-15
- **Статус:** одобрен пользователем (визуально провалидировано в visual-companion: вариант B + полный перенос темы)
- **Область:** collapsed-вид строки задачи в списке (side-панель). Иконки (дуотон) и раскрытая деталь не меняются.
- **Связано:** продолжает недавний редизайн иконок задач в той же строке.

---

## 1. Проблема / текущее состояние

Строка (`.task-row`) — трёхколоночный flex: `[иконка] [средняя колонка flex:1] [правый кластер]`.

- Правый кластер `.task-meta` — **`position:absolute; right:12px; top:50%`** ([sidepanel.css:1644-1654](../../sidepanel.css)), вынут из потока и приклеен к правому краю по центру строки.
- Чтобы текст не заехал под него, `.task-row` держит **`padding-right:56px`** ([sidepanel.css:1501](../../sidepanel.css)) — keep-out зона.
- Все мета-строки (время, `Сделка: …`, `Создал: …`, назначено, владелец, группа синхронизации) живут в средней колонке ([render.js:645-653](../../js/render.js)), ширину которой справа съедают кластер + 56px.

**Итог:** длинные `Сделка/Создал` переносятся/режутся рано, хотя справа под чипом — пустота. Горизонтальное место используется плохо.

## 2. Цели

1. Пустить мета-данные **под темой во всю ширину** (заезжая и под иконку, и под правый жёлоб) — без накладок.
2. Убрать зарезервированные 56px; место отдать контенту.
3. Тема видна **целиком** (полный перенос).

### Не-цели (YAGNI)

- Не трогаем иконки (дуотон-слот), раскрытую деталь (`.task-detail`), плеер записей, логику таймера.
- Не меняем, какие данные показываются — только их **расположение** в строке.
- Не меняем поведение раскрытия по клику и `stopPropagation` интерактивных контролов.

## 3. Дизайн — двухъярусная строка (вариант B)

`.task-row` становится **колонкой** из двух ярусов:

### Ярус 1 — `.task-row-head` (flex-строка, `align-items: flex-start`)
`[иконка/чекбокс 30px]  [Тема]  [правый контрол]`
- **Иконка/чекбокс** — существующий `checkboxHtml` (дуотон-слот / чекбокс личной задачи). Без изменений.
- **Тема** (`.task-text`) — `flex:1; min-width:0`, **полный перенос** (`white-space: normal; overflow-wrap: anywhere`), видна целиком; строка растёт по высоте.
- **Правый контрол** — основной элемент статуса/действия: чип (`.task-status-icon`) / ▷ запись / таймер заметки. `flex-shrink:0`, выровнен по верху, больше **не** `absolute`. Бейдж оставшегося времени сюда **не** входит — он уходит в ярус 2 (см. §5).

### Ярус 2 — `.task-row-meta` (во всю ширину, под иконку)
Мета-строки стопкой, **без левого отступа** (edge-to-edge под иконкой), порядок:
`время → Сделка/Контакт (related) → кто (назначено / владелец / Создал) → группа синхронизации`
- Сюда же переезжают: **бейдж оставшегося времени** (`.task-time-badge`, текст из `formatRemaining` — «N ч» / «N дн» / «N мин», без слова «осталось»; раньше был **справа**, внутри `.task-status-group`) и **время звонка/встречи** (`.task-row-time` — раньше **над темой** в средней колонке, не справа).
- Ярус 2 рендерится только если есть ≥1 мета-строка (иначе не создаётся — нет пустого зазора).

### Почему нет накладок
Правый контрол живёт только на ярусе 1 (рядом с темой). Всё во всю ширину — строго ниже (ярус 2, по потоку). Накладку раньше предотвращали 56px keep-out; теперь — порядок ярусов. `padding-right:56px` удаляется.

## 4. Правый контрол по типам задач

| Тип | Ярус 1 справа | Ярус 2 (мета) |
|---|---|---|
| CRM-задача | чип срока (выполнено/просрочено/без срока) | `2 ч · Сделка: … · Создал: …` (бейдж времени отделён от чипа, см. §5) |
| Звонок | ▷ кнопка записи / пусто (чип у звонков **не** показывается) | `12:30 · Сделка: …` |
| Встреча/Чат | пусто (чип у этих типов **не** показывается) | `время · Сделка: …` |
| Заметка | таймер (интерактивный play/pause + время) | `группа синхр. · назначено: …` |
| Личная задача | чекбокс — слева (в слоте); чип срока — справа, если есть | — (обычно пусто) |

«Один контрол» = один логический элемент. Исключение: **несколько записей** у звонка → существующий ряд ▷-кнопок (`.task-status-group` row); если он широк для узкой панели — добавить перенос на свою строку под темой (в CSS переноса пока нет; проверить визуально).

## 6. Взаимодействие
- Клик по строке разворачивает `.task-detail` — без изменений.
- Интерактивные контролы (чекбокс, ▷ запись, таймер, чип, кнопка мозга) сохраняют `stopPropagation` — не разворачивают строку.

## 7. Архитектура (границы)
Строка делится на два понятных юнита:
1. **Ярус 1 (head)** — интерактив + заголовок: иконка/чекбокс, тема, основной контрол.
2. **Ярус 2 (meta)** — только информационные строки, во всю ширину.

Граница чёткая: контролы и заголовок в head; справочная мета — в meta. Менять состав меты можно, не трогая head.

## 8. Краевые случаи
- **Длинная тема** — полный перенос, ярус 1 растёт; правый контрол top-align, остаётся у верхней строки темы.
- **Высокий правый контрол** (таймер заметки) — он на ярусе 1; мета всегда ниже → не пересекаются.
- **Несколько записей** — ряд ▷-кнопок (`.task-status-group` row); переноса в CSS пока нет — если при визуальной проверке вылезает за ширину, добавить перенос под тему.
- **Пустая мета** (личные задачи) — ярус 2 не рендерится.
- **Тёмная тема** — цвета/иконки не меняются.
- **Заметка с таймером** — таймер справа на ярусе 1, мета (группа/назначено) — ниже.

## 9. Тестирование
В репозитории нет тест-раннера; изменение визуально-разметочное → проверка **визуальная** (load unpacked, обе темы):
- Типы: CRM-задача (чип + мета), звонок с записью/без, встреча, чат, заметка с таймером, личная задача, просрочено/без срока/выполнено.
- Длинная тема (полный перенос), несколько мета-строк во всю ширину, несколько записей.
- Нет правого «жёлоба»-пустоты; нет накладок; раскрытие по клику работает.

## 10. Вне области
Иконки задач (дуотон) уже сделаны и не меняются; фикс полуночных уведомлений — отдельная задача; раскрытая деталь, формы, серверный API не затрагиваются.
