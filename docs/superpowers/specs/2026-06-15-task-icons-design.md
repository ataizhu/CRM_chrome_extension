# Дизайн: единый стиль иконок задач («Дуотон») + векторный «мозг»

- **Дата:** 2026-06-15
- **Статус:** одобрен пользователем (визуальная часть провалидирована в visual-companion)
- **Область:** иконки строки задачи в side-панели расширения (`crm/`)
- **Связано:** заменяет тяжёлый GIF-«мозг» (`assets/brain-process.gif`, 2.69 МБ)

---

## 1. Проблема / текущее состояние

Иконки строки задачи собраны из разных стилей и размеров:

- **Тип-иконки** (звонок/встреча/чат/заметка) — голые контурные SVG ~20px, цвет задаётся через `.task-activity-icon-*` (oklch-литералы), без подложки.
- **«В работе»** — два растровых `<img>` (статичный `brain-static.jpg` 45 КБ + анимированный `brain-process.gif` 2.69 МБ) в слоте 40px. Тяжело и выбивается по стилю.
- **Состояния** «выполнено»/«отменено» — контурные галочка/крестик 20px (`.task-status-slot`), крестик сейчас серый.
- **Вторичный чип срока** (`.task-status-icon`, 24px) — скруглённый **квадрат** с цветной заливкой; внутри **всегда глиф «часы»** независимо от состояния («выполнено»/«просрочено»/«без срока»). Глиф зашит статикой в `renderTaskItem` (render.js ≈609–612) и **не** подменяется на live-update (`refreshTaskTimeBadges` 78–115 переключает только `className`/`data-tooltip`).

Итог: разнокалиберно, один элемент весит 2.7 МБ, вторичный чип показывает один и тот же глиф «часы» во всех трёх состояниях.

## 2. Цели

1. Привести **все** иконки строки задачи к одному визуальному языку — «Дуотон» (глиф акцентного цвета на полупрозрачной круглой подложке того же цвета).
2. Заменить растровый «мозг» на инлайн-SVG + CSS-анимацию (вес ~1 КБ вместо 2.7 МБ).
3. Сделать палитру единым источником правды (CSS-токены), темизируемой под светлую/тёмную тему.

### Не-цели (YAGNI)

- Не меняем **логику** определения иконок (`eventStatusKind`, `activitytype`, `completed`/`end`) — только presentation.
- Не трогаем плеер записей, таймер заметок, кнопки действий, разметку `.task-detail`.
- Не вводим новые состояния задач и не меняем размеры/верстку самой `.task-row` (кроме унификации слота иконки).

## 3. Визуальная система «Дуотон»

Единый контейнер-слот для всех иконок строки:

- **Геометрия:** круг **30×30px**, `border-radius: 50%`, `flex-shrink: 0`, вертикальное выравнивание как у текущего слота (`margin-top: 1–2px`).
- **Подложка:** заливка акцентным цветом, **14%** прозрачности в светлой теме, **18%** в тёмной.
- **Глиф:** SVG **17×17px**, `fill: none`, `stroke: <accent>`, `stroke-width: 2`, `stroke-linecap/linejoin: round`.
- Обычный нативный чекбокс личных задач сохраняется как есть (~19px), но живёт внутри слота высотой 30px для общего ритма.

## 4. Палитра — CSS-токены (единый источник правды)

Определяются один раз в `:root`, переопределяются в `body.dark-theme`. Слот-классы потребляют только токены — хардкода цветов в слотах нет.

| Роль | Токен (stroke / bg) | Свет (stroke) | Тёмная (stroke) | Привязка |
|---|---|---|---|---|
| Звонок | `--icon-call` / `--icon-call-bg` | `#16a34a` | `#34d399` | `oklch(.55 .18 145)` / dark `oklch(.7 .17 150)` |
| Звонок без записи | используется `--icon-muted` + strike | серый | серый | как ниже «muted» |
| Встреча | `--icon-meeting` / `…-bg` | `#2563eb` | `#5b8def` | `oklch(.55 .2 250)` / `oklch(.68 .17 255)` |
| Чат | `--icon-chat` / `…-bg` | `#d97706` | `#e0a23c` | `oklch(.6 .15 85)` / `oklch(.72 .14 85)` |
| Заметка | `--icon-note` / `…-bg` | `#6b7280` | `#9aa1ac` | `--muted-foreground` |
| В работе (мозг) | `--icon-brain` / `…-bg` | `#7c3aed` | `#a78bfa` | новый токен |
| Выполнено | `--icon-done` / `…-bg` | `#16a34a` | `#34d399` | `--success` |
| Отменено | `--icon-cancel` / `…-bg` | `#dc2626` | `#f0635f` | `--destructive` |
| Просрочено (чип) | `--icon-overdue` / `…-bg` | `#dc2626` | `#f0635f` | `--destructive` |
| Без срока (чип) | `--icon-muted` / `…-bg` | `#6b7280` | `#9aa1ac` | `--muted-foreground` |

`*-bg` = тот же цвет на 14% (свет) / 18% (тёмная). Где уже есть системный токен (`--success`, `--destructive`, `--muted-foreground`) — переиспользуем его, а `--icon-*` делаем алиасом, чтобы палитра иконок читалась в одном месте.

**Каноничность значений:** источник правды — токены `--icon-*`. В таблице hex — целевое значение на роль/тему; oklch в колонке «Привязка» дан как ориентир на существующие токены, а **не** как второе независимое значение. Новые токены типа (звонок/встреча/чат) **заменяют** прежние oklch-литералы в `.task-activity-icon-*` (включая тёмный `oklch(.65 .18 145)` для звонка) — старые литералы удаляются, чтобы не осталось двух источников цвета. При реализации зафиксировать одно каноническое значение на ячейку (hex → пересчёт в oklch при желании, но одно значение).

**Замечание по красному:** «просрочено» и «отменено» оба красные. Конфликта нет — состояния взаимоисключающие на одной строке (просрочено = активная задача, отменено = терминальная), глифы и позиции разные (часы справа / крестик слева). Красный = «негатив/внимание».

## 5. Спецификация по элементам

Логика выбора слота в `renderTaskItem` **не меняется**; меняется только разметка/классы слота и CSS. Карта «состояние → слот»:

### 5.1 Тип-слоты (левый слот)
- **Call** → круг `--icon-call`, глиф «телефон».
- **CallNoRecording** (`activitytype==='Call'` и нет URL записи) → круг `--icon-muted` (серый), глиф «телефон» + диагональная линия-strike (сохранить текущую семантику «запись не состоялась»).
- **Meeting** → круг `--icon-meeting`, глиф «календарь».
- **Chat** → круг `--icon-chat`, глиф «облако».
- **Note** (`group==='Заметки'`) → круг `--icon-note`, глиф «документ».

**Куда вешать цвет круга:** сейчас слот-`<span>` (`.task-activity-type-slot` / `.task-note-type-slot`) **не несёт** класса типа — цвет берётся из класса внутреннего SVG (`.task-activity-icon-call` и т.д., render.js 363–371). Для дуотон-подложки нужен хук на самом слоте. Решение: повесить на слот модификатор типа (`.task-activity-type-slot.is-call|is-meeting|is-chat|is-call-no-rec`, `.task-note-type-slot`), а `background` (подложка) и `color` (stroke глифа через `currentColor`) брать из соответствующего `--icon-*`. Прежние `.task-activity-icon-*` oklch-классы удаляются — их роль (цвет) переезжает на слот/токены.

### 5.2 Состояние-слоты (левый слот)
- **In-work** (`eventStatusKind==='in_work'`) → круг `--icon-brain`, инлайн-SVG «мозг» + halo (см. §6).
- **Held / Выполнено** (`==='held'`) → круг `--icon-done`, глиф «галочка».
- **Not-held / Отменено** (`==='not_held'`) → круг `--icon-cancel` (красный), глиф «крестик». **Намеренная смена семантики цвета:** сейчас крестик серый (`.task-status-cross` → `--muted-foreground`, CSS 1434–1442); по выбору пользователя становится красным. Это сознательное решение, не регресс.
- **Обычная личная задача** → нативный чекбокс (без круга-подложки), выравнивание по слоту.

### 5.3 Вторичный чип срока (`.task-status-icon`, правая часть)
Приводится к дуотон-кругу **24×24px**. Сейчас внутри **всегда «часы»**; делаем глиф **состояние-зависимым**:
- **Completed** → `--icon-done`, глиф «галочка».
- **Overdue** → `--icon-overdue`, глиф «часы».
- **No-deadline** → `--icon-muted`, глиф «минус/тире».

Логика показа/скрытия сохраняется, но **глиф теперь зависит от класса**, поэтому его нужно синхронизировать в **двух** местах:
1. **Инициализация** `renderTaskItem` (≈609–612) — вместо зашитых «часов» подставлять глиф по `statusClass`.
2. **Live-update** `refreshTaskTimeBadges` (78–115) — сейчас меняет только `className`/`data-tooltip`; теперь должен **также подменять внутренний SVG** под новое состояние. Иначе класс и глиф рассинхронизируются (напр. задача стала просроченной «на лету»: класс `overdue`, а глиф остался «галочка»).

Чтобы не дублировать SVG, маппинг `statusClass → глиф` выносится в одну функцию-хелпер (напр. `statusIconGlyph(statusClass)`), используемую и в инициализации, и в live-update. Это интерфейс отдельного мини-юнита (см. §9).

## 6. «Мозг»: вектор + анимация

Замена обоих `<img>` на инлайн-SVG (две зеркальные path Lucide «brain») внутри дуотон-круга `--icon-brain` плюс `<span class="brain-halo">`. Каноническая разметка глифа (viewBox `0 0 24 24`, провалидирована в `icons-in-context.html`):

```html
<span class="brain-halo" aria-hidden="true"></span>
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" aria-hidden="true">
  <path d="M12 5a3 3 0 1 0-5.997.125 4 4 0 0 0-2.526 5.77 4 4 0 0 0 .556 6.588A4 4 0 1 0 12 18Z"/>
  <path d="M12 5a3 3 0 1 1 5.997.125 4 4 0 0 1 2.526 5.77 4 4 0 0 1-.556 6.588A4 4 0 1 1 12 18Z"/>
</svg>
```

- **Анимация:** `@keyframes halopulse` — подложка-halo плавно «дышит» (`scale .78→1`, `opacity .4→1`, `1.7s ease-in-out infinite`). Включается классом `.in-progress`.
- **Два режима** (как сейчас static/animated swap):
  - Кнопка «Начать работу» (`.task-brain-btn`, кликабельна, задача ещё не в работе) → **статичный** мозг, без пульса; сохранить `:hover`-подсветку.
  - Задача в работе (`.in-progress`, в т.ч. view-only `.task-brain-slot`) → **пульс**.
- Удаляются файлы `assets/brain-process.gif` и `assets/brain-static.jpg`, константы `brainStaticUrl`/`brainAnimatedUrl` и старые `.brain-icon`-правила.

## 7. Доступность и движение

- Декоративные SVG — `aria-hidden="true"`; существующие `title`/`aria-label` на слотах/кнопках сохраняются (звонок, «в работе», «выполнено», «отменено», тултипы чипа срока).
- Добавить `@media (prefers-reduced-motion: reduce)` — отключает `halopulse` (halo статичен). Раньше анимация была неотключаемой (GIF).
- Контраст stroke на подложке проверен в обеих темах на макете `icons-in-context.html`.

## 9. Архитектура (границы)

Три независимых, отдельно понятных юнита:

1. **Палитра-токены** (`:root` / `body.dark-theme`) — единственный источник цветов; интерфейс = имена `--icon-*`.
2. **Презентация слотов** (CSS-классы) — потребляют токены, ничего не знают о логике; интерфейс = классы слотов.
3. **Маппинг состояния → класс/глиф** (`render.js`) — без изменений в логике определения состояния; отдаёт нужный класс/глиф по `eventStatusKind`/`activitytype`/`completed`/`end`. Сюда входит чистый хелпер `statusIconGlyph(statusClass) → SVG-строка`, общий для инициализации (`renderTaskItem`) и live-update (`refreshTaskTimeBadges`), — единственный источник соответствия «класс чипа → глиф», чтобы они не расходились.

Палитру можно менять, не трогая render.js; render-логику — не трогая CSS; маппинг глифа чипа — в одном хелпере, а не продублирован в двух функциях.

## 10. Краевые случаи

- **CallNoRecording** — серый круг + strike, не зелёный.
- **CRM-группа «только просмотр»** (`CRM_GROUP_NAME`) — те же дуотон-иконки, но без интерактива (мозг не кликабелен).
- **Заметка с таймером** — иконка-документ слева не конфликтует с бейджем таймера справа.
- **Тёмная тема** — токены `body.dark-theme` дают более светлые stroke и 18% подложку.
- **Reduced-motion** — halo не пульсирует.
- **Live-update чипа срока** — при смене состояния «на лету» класс и глиф должны меняться **вместе** (см. §5.3, п.2), иначе визуальный рассинхрон класс↔глиф.
- **Двойной сигнал «выполнено»** (личная задача: чекбокс-✓ слева и чип «выполнено» справа) — поведение существующее, не вводим регресс, оставляем как есть.

## 11. Тестирование

Изменение визуально-разметочное; ценность глубоких тестов низкая. Минимальный объём:

- Юнит-тест `renderTaskItem` (Vitest): для набора типовых задач (call с записью / call без записи / meeting / chat / note / held / not_held / in_work / personal-todo) проверить, что вывод содержит правильный класс слота и **не содержит** ссылок на `brain-process.gif`/`brain-static.jpg`/`<img`.
- Ручной визуальный чек-лист: обе темы, состояния из §5, пульс «в работе», reduced-motion.

## 12. Развёртывание

- Клиентское изменение → требует пересборки `pack-extension.sh` и переаплоада в стор (как и прочие правки клиента).
- Репозиторий git не инициализирован — дизайн-док и код не коммитятся; при желании версионировать можно `git init` отдельным шагом.

## 13. Вне области

Серверный PHP-API, синхронизация групп, ротация ключей Supabase, разделение `ui.js` — отдельные задачи, этим дизайном не затрагиваются.
